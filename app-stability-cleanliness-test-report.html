This pass added focused regression coverage for selected-provider model metadata so stale capabilities cannot silently hide tools. It also tightened production typing in chat session, markdown, completion, thread, model-selection, updater, global app bridge, and message-rendering paths. What improved

  • Fixed first chat send: initial messages are no longer consumed before the delayed send runs.
  • Fixed app close behavior: normal close is no longer intercepted and recreated like a reload.
  • Fixed tools visibility: MCP tools are visible when connected, even if model capability metadata is stale.
  • Hardened selected model resolution: active provider metadata now wins over stale selected-model cache data.
  • Reduced dead code: removed unused Rust app-token state and tightened several TypeScript types.
  • Reduced production type looseness: tightened chat session, markdown renderer, completion content, thread research, and dropdown model typing.
  • Cleaned remaining hand-written any usage: production any/cast scan now only reports generated route tree casts.
  • Updated regression tests: toolbar, startup, provider, hardware, MCP, and model lifecycle paths now have stronger coverage.
  • Clean validation: lint, Rust tests, full JS/TS tests, and module coverage gates pass.

Remaining gaps before calling it fully stable

Area Current risk Why it lowers the score Recommended next step
Real startup flow Tauri startup, extension install, local API bootstrap, MCP startup, model loading, and frontend boot are complex. Unit tests pass, but only real app/E2E tests prove the full startup path. Add a smoke E2E test that opens the app and verifies shell, icon, chat, and service hub readiness.
Local knowledge / MCP MCP transport can close or fail to list tools during long sessions. This affects fabric search/extract reliability and user trust in local knowledge answers. Add recovery tests and UI-visible source/tool-call confirmation for knowledge searches.
Local model lifecycle Engine availability depends on llamacpp process startup, readiness, and model compatibility. Failures here create “engine not available” or slow/hanging load states. Add integration tests around engine readiness, failure recovery, and timeout behavior.
UI regression coverage Containers and routes have lower measured coverage than hooks/services/lib. Visual or state regressions can still slip through even when unit tests pass. Add Playwright-style flows for chat history persistence, model selection, local knowledge, and hub links.
Test output noise Some tests intentionally print warnings/errors while passing. Noisy output makes real failures harder to notice quickly. Silence expected warnings in tests with scoped spies where safe.
Generated route casts The remaining production any scan only reports generated TanStack route tree casts. Generated files should not be manually edited; they do not represent hand-written app cleanliness debt. Regenerate route types from the routing toolchain when the generator supports stricter output.
